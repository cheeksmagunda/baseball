Migration parity diff — Step 4 (CSV-path vs SQLite-path readers)
================================================================

Each of the 7 calibration / audit / validate scripts was run twice:
  $ BO_HISTORICAL_SOURCE=csv    python scripts/<reader>.py [args]
  $ BO_HISTORICAL_SOURCE=sqlite python scripts/<reader>.py [args]

The SQLite path materialises data/historical.db into a tempdir via
scripts/export_historical_csvs.export_all() and rebinds the data file
paths inside main() to that tempdir.  The CSV path reads from /data/
directly.  Both produce byte-identical stdout because data/ files are
the canonical export of data/historical.db (Step 2 byte-stable contract).
